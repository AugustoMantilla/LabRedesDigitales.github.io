Redes de computadora con Switches y Cables

Augusto Mantilla Molina
Ingeniería Mecatrónica
Jorge Eduardo Limón Balderas
Ingeniería Mecatrónica
Fecha: Abril 23, 2025
Switches Router

- Resumen -

El propósito de esta práctica es conocer, identificar y comprobar el funcionamiento de una red de área local (LAN) a través del uso de dispositivos de interconexión como switches y routers. A lo largo de la actividad, se configuraron diferentes topologías de red que permitieron observar el comportamiento de la comunicación entre varios equipos conectados en red, tanto de manera alámbrica como inalámbrica. Se implementaron redes separadas utilizando un switch y un router, conectando al menos tres computadoras y un servidor web, con asignación automática de direcciones IP.

- Introducción -

Las redes de área local permiten la interconexión de múltiples dispositivos dentro de un entorno limitado, facilitando el intercambio de información, el acceso a servicios y el uso compartido de recursos. Los dispositivos de interconexión como switches y routers desempeñan un papel esencial, ya que gestionan el tráfico de datos y permiten la comunicación eficiente entre los equipos conectados. Se configuraron diferentes tipos de red —alámbrica, inalámbrica y combinada— integrando varios dispositivos, incluidos un servidor web y computadoras con direcciones IP asignadas automáticamente.

- Materiales -

  • 6 dispositivos
  • Cables directos
  • Cables cruzados
  • 1 switch
  • 1 router

- Desarrollo -

Para comenzar, se construyó una red utilizando únicamente un switch como dispositivo de interconexión. Se conectaron tres computadoras y un servidor web mediante cables Ethernet directos, asegurando que todos los dispositivos estuvieran físicamente enlazados al switch. Para verificar la correcta configuración de la red, se utilizó el comando ipconfig en cada equipo con el fin de identificar sus direcciones IP y confirmar que se encontraban dentro del mismo rango de red. Posteriormente, se utilizaron comandos ping desde cada computadora hacia las demás y al servidor para comprobar la conectividad.

Despues, se trabajó con una red centrada en un router. El servidor que se utilizó en la red anterior se conectó directamente al router mediante un cable de red, mientras que tres dispositivos móviles se unieron a la red de manera inalámbrica. Desde el servidor se realizó la verificación de conectividad hacia los dispositivos móviles usando comandos ping, observando una respuesta satisfactoria.

Para finalizar, se procedió a integrar las dos redes previas, conectando el switch al router mediante un cable directo o cruzado, según la configuración de los puertos. Esta conexión permitió unificar los dispositivos conectados por cable al switch con los dispositivos móviles conectados al router de forma inalámbrica. Al establecer esta conexión, se observó que las direcciones IP de las computadoras cambiaron, adaptándose a la nueva configuración general de red administrada por el router. Desde el servidor se verificó la conectividad hacia los seis dispositivos conectados, utilizando nuevamente comandos ping. Por último, se comprobó que todos los dispositivos, tanto alámbricos como inalámbricos, podían acceder simultáneamente a la página web alojada en el servidor.

- Conclusiones -

La práctica realizada permitió afianzar los conocimientos teóricos y prácticos sobre la construcción y funcionamiento de redes de área local mediante el uso de switches y routers. La asignación automática de direcciones IP, la selección adecuada de cables y la verificación de la conectividad con herramientas como el comando PING y el acceso a un servidor web fueron elementos clave para validar el correcto funcionamiento de la red. La conexión exitosa de dispositivos alámbricos e inalámbricos dentro de un mismo entorno de red demostró la importancia de una configuración estructurada y bien planificada.
